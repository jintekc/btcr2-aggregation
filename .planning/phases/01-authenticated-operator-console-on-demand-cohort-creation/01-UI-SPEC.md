# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

This phase adds the **first authenticated operator surface** on top of an existing, coherent custom design system. It introduces: a login screen, a login-gated operator console (cohort create form + draft review + operator cohort list), the two-step create-draft-then-advertise flow, and a minimal anonymous public status element. The contract below EXTENDS the established system rather than replacing it - the existing dark-slate + bitcoin-orange theme and the in-house primitives (`packages/web/src/ui/primitives.tsx`) are the source of truth and MUST be reused, not re-invented.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | in-house primitives (`packages/web/src/ui/primitives.tsx`: `Card`, `Button`, `Badge`, `StatusDot`, `SectionTitle`, `Mono`, `CopyField`) over Tailwind CSS v4 |
| Icon library | none — no icon font/SVG set. Status is conveyed by `StatusDot` (colored dot + optional live pulse) and `Badge` pills. Do NOT introduce an icon dependency this phase. |
| Font | Body/UI: `ui-sans-serif, system-ui, -apple-system, "Segoe UI", sans-serif`. Mono: `--font-mono` (`ui-monospace, "SF Mono", "JetBrains Mono", "Fira Code", monospace`) for DIDs, hashes, cohort IDs, txids. |

**shadcn gate outcome:** intentionally NOT initialized. A mature custom design system already exists (Tailwind v4 `@theme` tokens in `packages/web/src/index.css` + typed primitives). The gate precondition "no design system detected" is false, and the project constraint (CLAUDE.md: stack is "established, do not churn without reason") makes adding shadcn a churn with zero requirement coverage. Extend the existing primitives instead.

**Reuse map (existing → Phase 1 use):**
- `Button` variants `primary` / `ghost` / `danger` → advertise & sign-in (primary), create-draft & cancel (ghost), discard-draft (danger).
- `Card` → login panel, create form, each cohort row/card, public status card.
- `Badge` + `StatusDot` tones (`neutral`/`accent`/`good`/`warn`/`bad`) → draft vs advertised state (draft = `neutral`, advertised = `accent`).
- `SectionTitle` → form section + console section headings (uppercase micro-label style).
- `CopyField` / `Mono` → cohort IDs and any copyable identifiers in the operator list.
- New primitives needed (add to `ui/primitives.tsx`, matching the file's conventions): a text `Input`, a `Select`, and a `Field` label+control+error wrapper. These do not exist yet and are the only additions permitted.

**New route:** the operator console lives at **`/operator`** (clean operator/service/aggregator framing per D-20; deliberately NOT `/admin`). The anonymous participant experience stays at `/`. Server-side session enforcement (D-04) is the real boundary; the route/bundle split is only presentation.

## Spacing Scale

Declared values (Tailwind v4 defaults, all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text and inline gaps (`gap-1`) |
| sm | 8px | Compact element spacing, badge/dot gaps (`gap-2`) |
| md | 16px | Default element spacing, card padding (`p-4`, `gap-3`≈12px for tight rows) |
| lg | 24px | Section padding, header margin (`py-6`, `mb-6`) |
| xl | 32px | Layout gaps between major console blocks |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Card inner padding: `p-4` (16px) for list rows, `p-5` (20px) for the login panel and create form (matches existing `RegisterPanel`). Vertical rhythm inside cards: `space-y-4` (16px).

Exceptions (inherited from existing primitives — do not "fix", match them for consistency):
- Pill/`Badge` padding `px-2.5 py-0.5` (10px / 2px).
- Micro-caption padding `px-3 py-2` on inset chips/banners; small status banners `px-2.5 py-1.5`.
- Touch target: interactive buttons keep `px-4 py-2` (min height ~36px). Any icon-only control this phase must be ≥44px square — none are currently planned.

## Typography

Canonical roles (4 sizes, weights 400 + 600, with a documented exception):

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 | 1.5 |
| Label | 12px (`text-xs`, uppercase, `tracking-[0.14em]`) | 600 | 1.4 |
| Heading | 20px (`text-xl`) | 700 | 1.2 |
| Display | 28px (`text-3xl`) | 700 | 1.15 |

- **Display (28px/700):** login screen title and the operator console landing heading only.
- **Heading (20px/700):** the app name / in-app section headers (matches existing `App.tsx` h1).
- **Label (12px/600 uppercase):** form field labels and `SectionTitle` headings.
- **Body (14px/400):** all descriptive/help/instructional copy.

Weight rule: **regular (400)** and **semibold (600)** are the two canonical weights. **Bold (700)** is a documented exception, reserved for the Display/Heading roles and stat counters (`tabular-nums` numeric values) only — never for body or label runs. Do not introduce `font-medium` (500) in new code; use 600 for emphasis.

Micro-caption exception: 10-11px uppercase captions (`text-[0.6rem]`/`text-[0.65rem]`, e.g. `CopyField` labels, counter captions) are inherited and permitted for dense secondary metadata only. Mono runs render at `~13px` (`text-[0.8rem]`).

## Color

Existing `@theme` tokens (OKLCH) in `packages/web/src/index.css` — dark slate canvas, single bitcoin-orange accent. Reuse the token names (`bg-canvas`, `bg-surface`, `text-accent`, `border-edge`, …); do NOT hardcode hex.

| Role | Value (token) | Usage |
|------|-------|-------|
| Dominant (60%) | `oklch(0.17 0.015 260)` (`--color-canvas`) | Page/app background, inset field backgrounds |
| Secondary (30%) | `oklch(0.21 0.018 260)` (`--color-surface`), `oklch(0.26 0.02 260)` (`--color-surface-2`) | Cards, login panel, nav, form controls, list rows |
| Accent (10%) | `oklch(0.74 0.16 55)` (`--color-accent`) | See reserved list below |
| Destructive | `oklch(0.68 0.2 25)` (`--color-bad`) | Destructive actions + error banners only |

Text tiers: `--color-ink` (primary), `--color-muted` (secondary/body help), `--color-faint` (captions). Borders: `--color-edge` / `--color-edge-strong`.

**Accent (bitcoin orange) reserved for — explicit list, nothing else:**
- The primary action button (`Advertise cohort`, `Sign in`) — `bg-accent text-accent-ink`.
- The active nav/route indicator (operator vs public).
- The accent word in the product wordmark (`aggregation`).
- Links (underlined, `text-accent`).
- Live/in-progress affordances: `StatusDot` accent tone + `.pulse` ring (e.g. a cohort transitioning through keygen/signing).

Accent is NOT applied to: all buttons, all borders, form focus rings on non-primary controls, or the advertised-state badge background beyond the existing `bg-accent/15` tint. Draft state = `neutral` tone; advertised (open, joinable) = `accent` tone.

**Destructive (`--color-bad`) reserved for:** the `Discard draft` button (`Button variant="danger"`), inline error banners (`border-bad/40 bg-bad/10 text-bad`), and the inherited mainnet REAL-FUNDS warning. Never for advertise/close/ordinary operator actions.

Semantic `--color-good` (success) and `--color-warn` (attention) remain available for status only (e.g. a successfully advertised confirmation), not as general UI color.

## Copywriting Contract

Clean operator/service/aggregator framing (D-20). NO "booth" / "attendee" / "demo" wording in any new Phase 1 string. Sentence case for buttons and copy; the operator is addressed directly ("you"/"your service").

| Element | Copy |
|---------|------|
| Primary CTA | `Advertise cohort` |
| Empty state heading | `No cohorts yet` |
| Empty state body | `Create a cohort to advertise it into this service's directory.` |
| Error state | `Incorrect password. Check the operator password set for this service and try again.` |
| Destructive confirmation | Discard draft: `Discard this draft? It hasn't been advertised, so nothing has been published to the directory yet.` — confirm button `Discard draft`, cancel `Keep draft`. |

Full string set for this phase:

**Sign in / auth**
- Login screen title (Display): `Operator console`
- Login help (Body): `Sign in with this service's operator password to create and advertise cohorts.`
- Password field label: `Operator password`
- Sign-in button (primary): `Sign in`
- Sign-in in-progress: `Signing in…`
- Auth error (invalid password): `Incorrect password. Check the operator password set for this service and try again.` (never reveal whether a session/account exists — wrong-credential message only)
- Fail-closed notice (no `OPERATOR_PASSWORD` set at boot, D-07): heading `Operator console is disabled` / body `This service booted without an operator password, so the console is turned off. Set OPERATOR_PASSWORD and restart the service to enable operator sign-in.`
- Sign out button (ghost): `Sign out`

**Create cohort form (SVC-01)**
- Section title: `Create a cohort`
- Beacon type field label: `Beacon type` — options `CAS` / `SMT`
- Threshold field label: `Co-sign threshold (n-of-n)`
- Capacity field label: `Capacity (max seats)`
- Active network (read-only display, D-10): `Network: {activeNetwork}` shown as a `Badge` — the service's single active network, not an editable control this phase.
- Create button (ghost, non-destructive): `Create draft`
- Create-in-progress: `Creating…`
- Validation error (capacity < threshold): `Capacity must be at least the co-sign threshold.`
- Validation error (threshold < 1): `Threshold must be at least 1 signer.`

**Draft review → advertise (SVC-02)**
- Draft badge: `Draft` (neutral tone)
- Advertised badge: `Advertised` (accent tone)
- Advertise button (primary): `Advertise cohort`
- Advertise-in-progress: `Advertising…`
- Advertise success (transient, good tone): `Advertised — now joinable in the directory.`
- Discard button (danger): `Discard draft`

**Operator cohort list (D-16)**
- Section title: `Your cohorts`
- Empty heading: `No cohorts yet`
- Empty body: `Create a cohort to advertise it into this service's directory.`
- Per-row fields (Body/Mono): network, beacon type, seats `{joined}/{capacity}`, state badge.

**Public status surface (anonymous, D-09)**
- Service-up label (good `StatusDot`): `Service online`
- Active network chip: `{activeNetwork}` (reuse existing network badge; keep the mainnet `· REAL FUNDS` bad-tone variant).
- Open-cohort count: `{n} open cohorts` — when `n === 0`: `No open cohorts right now`.
- Public directory empty body: `This service isn't advertising any cohorts right now. Check back soon.`
