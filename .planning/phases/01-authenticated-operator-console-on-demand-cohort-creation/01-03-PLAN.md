## Phase Goal (user story)

**As a** service operator, **I want to** advertise a configured draft into my service's cohort directory (and stop the process from perpetually auto-advertising cohorts I never asked for), **so that** the only cohorts my service offers are the ones I explicitly advertised, they show up as open and joinable in the directory a participant can browse, and anyone can see a truthful open-cohort count.

<objective>
Complete the two-step flow (SVC-02, D-12) and retire the demo driver: an authenticated operator advertises a draft (the sole caller of `runner.advertiseCohort` now), a public `GET /v1/directory` derives the open-cohort set from the live `runner.session.cohorts` (one source of truth, D-15), a public `GET /v1/status` exposes up/network/open-count (D-09), the boot-time perpetual auto-advertise loop and boot-path fillers are removed (D-17/D-18), and the web console gains an Advertise action + the anonymous public status element.

Purpose: A fresh self-hosted service advertises nothing until the operator acts, and an advertised cohort appears as an open joinable directory entry - directly proving ROADMAP success criterion 3 and setting up Phase 2's browse UI and Phase 5's pause/cancel.

Output: `advertiseDraft` + `directory` + `status` in `operator-cohorts.ts`, their routes in `hono-adapter.ts`, the loop/fillers removal in `demo-server.ts`, extended `operator-cohorts.spec.ts`, and the web Advertise button + `PublicStatus`.
</objective>
