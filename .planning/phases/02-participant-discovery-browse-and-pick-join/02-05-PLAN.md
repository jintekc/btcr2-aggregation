## Phase Goal (user story)

**As a** participant browsing a service's directory, **I want to** see a seat count and co-sign figure that tell the truth (n seats that all fill, co-signed n-of-n), **so that** I am not misled by a phantom "open" seat that can never fill because the cohort locks the instant it reaches the co-sign threshold. (Closes UAT finding F1a/F1b under the PART-01 directory-faithfulness gap.)

<objective>
Close UAT findings F1a + F1b by collapsing the operator's cohort-size model to a single number n (F1b), which makes the participant directory honest for free (F1a). Today the operator sets a `threshold` (-> `minParticipants`) and a separate `capacity` (-> `maxParticipants`), the library finalizes at `minParticipants`, and `maxParticipants` is only an accept ceiling that is never reached - so a `2/3` cohort locks and co-signs 2-of-2 while the directory advertises a 3rd seat (`0/3 seats, 3 open`) that can never fill. The authoritative user decision is that capacity must NEVER exceed threshold: collapse to a SINGLE `Cohort size (n-of-n)` field, enforce `min == max == n` on both the browser AND the server so `capacity > threshold` is unrepresentable, and let the existing CohortRow (`{joined}/{capacity} seats`, `Co-sign: {threshold}-of-{threshold}`) become truthful with zero display change.

Purpose: the participant directory faithfully represents the cohort's signing set and seat expectations (the PART-01 gap truth), and the operator can no longer draft a cohort that misrepresents itself.

Output: the collapsed `DraftInput { beaconType, size }` + `validateDraft` + `createDraft` (min == max == n) in `operator-cohorts.ts`, the updated body-shape 400 string in `hono-adapter.ts`, the single-size `CreateCohortForm`, the `DraftInput` type in `lib/operator.ts`, and the two e2e create bodies switched to `{ beaconType, size }`.
</objective>
