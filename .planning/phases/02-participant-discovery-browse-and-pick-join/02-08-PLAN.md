## Phase Goal (user story)

**As a** participant browsing a service's directory, **I want to** see a cohort's real shape - n seats that all join (the cohort starts only when n join) and a separate signing threshold k that k of the n must reach to anchor - **so that** I understand both what I am committing to (co-signing together) and the liveness floor, instead of a single collapsed number that hides the operator's threshold control. (Closes UAT gap G-02-1 under the PART-01 directory-faithfulness truth.)

<objective>
Close UAT gap G-02-1: restore the operator's signing-threshold control that plan 02-05 over-corrected away, as a truthful TWO-FIELD k-of-n model. 02-05 collapsed the cohort to a single n-of-n number to kill the F1b phantom seat; that fix stands (every seat still fills at finalize-at-n), but it also deleted the operator's ability to set a signing floor k < n. The plumbing for k already exists (02-07 shipped `buildCohortConfig`'s optional `fallbackThreshold` and `autoFallbackOnStall`, and beacon-address.ts already commits the fallback leaf), it is simply not exposed on the create form, not carried as a distinct k in the DTOs, and not displayed (CohortRow renders `Co-sign: {threshold}-of-{threshold}` = n-of-n).

The target model (user-confirmed at UAT Test 1, fully designed + adversarially verified in 02-KOFN-DESIGN.md, which the executor MUST treat as authoritative):

1. Cohort size n (seats): `minParticipants == maxParticipants == n`, kept VERBATIM from 02-05 (T-KOFN-04). The cohort does not finalize until all n join.
2. Signing threshold k, `1 <= k <= n`: `fallbackThreshold = k` on the CohortConfig. The optimistic PRIMARY spend stays n-of-n MuSig2 (all n co-sign the cheap key path). If that round STALLS mid-signing, the ADR-042 k-of-n script-path fallback completes as long as at least k of the n sign. There is NO genuine k-of-n PRIMARY in @did-btcr2/aggregation@0.4.0; k is the fallback floor.

Directory honesty: `joined/n seats` + a `k-of-n` co-sign figure (DTO `capacity = n`, `threshold = k`). This restores the second number 02-05 removed, with correct semantics - the second number is the signing floor k, NOT the removed phantom `maxParticipants` ceiling. No F1b regression: every seat still fills (finalize-at-n), so both numbers stay truthful.

Beacon-address honesty note (Decision 2, verifier-corrected - do NOT claim byte-identical): today's 4-arg createDraft leaves the fallback leaf at the library's implicit n-1, so a DEFAULT (k == n) cohort's committed beacon address CHANGES (n-1 leaf -> n leaf). This is deliberate: it closes a pre-existing honesty gap where the UI said "all signers required" while the committed script tree let n-1 anchor. It is safe because no addresses are persisted, the fixture recomputes from config on both sides, LIVE derives fresh addresses, and no e2e asserts a specific address. Document this in the createDraft comment.

Output: the two-field `DraftInput { beaconType, size, threshold? }` + `validateDraft` (normalize `?? size`, guard `[1, size]`, fallback-off guard) + `createDraft` (fallbackThreshold = k, DTO flip at all four emit sites) in operator-cohorts.ts, the `autoFallbackOnStall` threading through createOperatorCohorts, the two-field create form + `k-of-n` display, and a NEW n=4/k=2 hermetic capstone proving both the upper path and the k floor.
</objective>
