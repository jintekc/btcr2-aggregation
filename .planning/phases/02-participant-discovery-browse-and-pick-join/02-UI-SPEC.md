# Phase 2 ‚Äî UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

This phase turns the participant landing at `/` into a **discover-first browse surface**: a service-identity header plus a polled directory of the service's open cohorts, from which the participant picks one, reveals an inline identity step, and joins by choice - replacing today's KeyGen-first linear stepper entry and the `shouldJoin: async () => true` auto-accept. It EXTENDS the established, locked design system (custom Tailwind v4 `@theme` tokens in `packages/web/src/index.css` + the in-house primitives in `packages/web/src/ui/primitives.tsx`); it does NOT introduce a new one. Per CONTEXT D-11/D-13 the existing submit/co-sign/resolve tail (`RegisterPanel`/`PublishPanel`/`ResolvePanel`/`ResultCard`) is reused **unchanged** after seating; Phase 3 rewires that tail and is out of scope here.

**shadcn gate outcome:** intentionally NOT initialized (unchanged from Phase 1). A mature custom design system already exists (Tailwind v4 `@theme` tokens + typed primitives). The gate precondition "no design system detected" is false, and the project constraint (CLAUDE.md: stack is "established, do not churn without reason") makes adding shadcn a churn with zero requirement coverage. Extend the existing primitives instead.

**New primitives needed this phase: NONE.** The browse list, per-cohort row/card, service-identity header, inline identity step, and seated confirmation all compose from primitives that already exist (see reuse map). If planning discovers a genuine gap, any new primitive MUST be added to `packages/web/src/ui/primitives.tsx` matching that file's conventions and justified in the plan - but the intent of this phase is zero new primitives.

**Reuse map (existing ‚Üí Phase 2 use):**
- `Card` ‚Üí the service-identity header, each cohort row/card, the inline identity panel, the seated confirmation, and the empty/unreachable/failed-join state panels.
- `Badge` + `StatusDot` tones (`neutral`/`accent`/`good`/`warn`/`bad`) ‚Üí plain-language cohort status label (Open = `accent`, Filling = `warn`, Collecting updates = `neutral`, Full = `neutral` muted), beacon-type chip, active-network chip. The header service-online indicator reuses `StatusDot tone="good" pulse` (as in `PublicStatus.tsx`).
- `Button` variants ‚Üí `primary` for the join CTA (row `Join` and inline `Join cohort`), `ghost` for `Cancel` / `Leave cohort`, `danger` is NOT used this phase (no destructive participant action - leaving a not-yet-full seat is benign, D-15/D-16).
- `SectionTitle` ‚Üí the directory section heading and the identity-step heading (uppercase micro-label style).
- `Mono` / `CopyField` ‚Üí the cohort id (copyable) per row and in the seated confirmation.
- `Field` / `Input` / `Select` ‚Üí only if the import-identity path needs a labeled control; the generate-new-identity default reuses the existing `KeyGenPanel` and import panels from `packages/web/src/components/participant/*` (D-04).
- Existing participant tail components (`RegisterPanel`, `PublishPanel`, `ResolvePanel`, `ResultCard`, and the participant store's register/submit/co-sign/resolve logic) ‚Üí reached from the seated state, reused unchanged (D-11).
- `PublicStatus.tsx` polling pattern (10s `GET /v1/status`) and `lib/operator.ts` `fetchDirectory`/`fetchStatus` (public, `credentials: 'omit'`) ‚Üí the model + data source for the ~5s directory poll (D-05) and the service-identity header (D-02).

**Route:** the participant browse directory becomes the landing at **`/`** (D-13) - the front door replacing the standalone KeyGen-first entry. The operator console stays at `/operator` (Phase 1). Base URL is `window.location.origin` (same-origin, D-01); no URL-input field.

## Spacing Scale

Declared values (Tailwind v4 defaults, all multiples of 4 - identical to Phase 1, do not diverge):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps, dot-to-text (`gap-1`) |
| sm | 8px | Compact element spacing, badge/dot gaps (`gap-2`) |
| md | 16px | Default element spacing, card/row padding (`p-4`), directory row vertical rhythm (`space-y-4`) |
| lg | 24px | Section padding, header margin (`py-6`, `mb-6`) |
| xl | 32px | Layout gaps between the header, directory, and (post-seat) tail blocks |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Card inner padding: `p-4` (16px) for directory list rows, `p-5` (20px) for the service-identity header (matches `PublicStatus.tsx`) and the inline identity panel. Vertical rhythm inside multi-field cards: `space-y-4` (16px). Directory list gap between rows: `space-y-3` (12px) or `space-y-4` (16px).

Exceptions (inherited from existing primitives ‚Äî match them for consistency, do not "fix"):
- Pill/`Badge` padding `px-2.5 py-0.5` (10px / 2px); network chip `px-3 py-1`.
- `CopyField` inset padding `px-3 py-2`.
- Touch target: interactive buttons keep `px-4 py-2` (min height ~36px). No icon-only controls are planned this phase; if one is added it must be ‚â•44px square.

## Typography

Canonical roles (4 sizes, weights 400 + 600, with the same documented 700 exception as Phase 1):

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 | 1.5 |
| Label | 12px (`text-xs`, uppercase, `tracking-[0.14em]`) | 600 | 1.4 |
| Heading | 20px (`text-xl`) | 700 | 1.2 |
| Display | 28px (`text-3xl`) | 700 | 1.15 |

- **Display (28px/700):** the landing / directory page title only (a single focal heading, e.g. the service-identity header's primary line).
- **Heading (20px/700):** the product wordmark / in-app section headers; the seated-confirmation headline.
- **Label (12px/600 uppercase):** `SectionTitle` directory + identity-step headings, per-row micro-labels (e.g. "SEATS", "CO-SIGN"), `CopyField` labels.
- **Body (14px/400):** cohort descriptive fields, status help, empty/error copy, identity-step help.

Weight rule: **regular (400)** and **semibold (600)** are the two canonical weights. **Bold (700)** is the documented exception, reserved for the Display/Heading roles and numeric seat/threshold counters (`tabular-nums`) only ‚Äî never for body or label runs. Do not introduce `font-medium` (500) in new markup; the inherited `Badge` primitive uses `font-medium` internally and is left as-is, but new copy uses 600 for emphasis.

Micro-caption exception: 10-11px uppercase captions (`text-[0.6rem]`/`text-[0.65rem]`, e.g. `CopyField` labels) are inherited and permitted for dense secondary metadata only. Mono runs (cohort id) render at `~13px` (`text-[0.8rem]`, via `Mono`).

Focal point: the browse landing has a single clear focal heading (Display/28px on the service-identity header), then the directory list; the inline identity step and the seated confirmation each have one Heading-level focal line so the participant always knows the current state at a glance.

## Color

Existing `@theme` tokens (OKLCH) in `packages/web/src/index.css` ‚Äî dark slate canvas, single bitcoin-orange accent. Reuse the token names (`bg-canvas`, `bg-surface`, `text-accent`, `border-edge`, ‚Ä¶); do NOT hardcode hex.

| Role | Value (token) | Usage |
|------|-------|-------|
| Dominant (60%) | `oklch(0.17 0.015 260)` (`--color-canvas`) | Page/app background, inset `CopyField`/input surfaces |
| Secondary (30%) | `oklch(0.21 0.018 260)` (`--color-surface`), `oklch(0.26 0.02 260)` (`--color-surface-2`) | The service-identity header card, each directory row/card, the identity panel, the seated card |
| Accent (10%) | `oklch(0.74 0.16 55)` (`--color-accent`) | See reserved list below |
| Destructive | `oklch(0.68 0.2 25)` (`--color-bad`) | Error/unreachable/failed-join banners only |

Text tiers: `--color-ink` (primary), `--color-muted` (secondary/body help), `--color-faint` (captions). Borders: `--color-edge` / `--color-edge-strong`.

**Accent (bitcoin orange) reserved for ‚Äî explicit list, nothing else:**
- The join CTA (`Join` on a row, `Join cohort` in the inline identity step) ‚Äî `bg-accent text-accent-ink`.
- The **Open (joinable)** cohort status `Badge` ‚Äî `accent` tone (mirrors Phase 1's advertised = accent; an open, joinable cohort is the accent-worthy affordance in the directory).
- The active nav/route indicator (participant `/` vs operator `/operator`), consistent with Phase 1.
- The accent word in the product wordmark (`aggregation`).
- Links (underlined, `text-accent`).
- Live/in-progress affordances only: `StatusDot` accent tone + `.pulse` ring for a cohort actively transitioning through keygen/signing (post-seat), and the reused service-online dot keeps its existing `good`+pulse treatment.

Accent is NOT applied to: every cohort row border, non-Open status badges, the seats/threshold counters, the cohort-id `Mono`/`CopyField`, form focus rings on non-primary controls, or the beacon-type/network chips. To keep the accent scarce in a list of many rows, only the **Open** status badge and the row's single **Join** button may carry accent per row; a `Full`/`Filling`/`Collecting updates` row shows a `Join` button that is disabled (`opacity-40`, the primitive's built-in disabled treatment) so accent recedes.

Status tones (plain-language labels, D-09):
- **Open** (raw `Advertised`, seats available) ‚Üí `Badge tone="accent"` ‚Äî joinable.
- **Filling** (raw `CohortSet`) ‚Üí `Badge tone="warn"` ‚Äî pre-signing, roster forming.
- **Collecting updates** (raw `CollectingUpdates`) ‚Üí `Badge tone="neutral"`.
- **Full** (client-derived when `joined >= capacity`, any open phase) ‚Üí `Badge tone="neutral"` (muted) + disabled `Join`.

**Destructive (`--color-bad`) reserved for:** the service-unreachable banner and the failed-join banner (`border-bad/40 bg-bad/10 text-bad`), plus the inherited mainnet `¬∑ REAL FUNDS` network-chip variant carried over from `PublicStatus.tsx`. No participant action this phase is destructive (leaving a not-yet-full seat is benign, D-15/D-16), so `Button variant="danger"` is NOT used.

Semantic `--color-good` (success) and `--color-warn` (attention) remain status-only: `good` for the service-online dot and a transient "Seated" success confirmation; `warn` for the Filling label. Not general UI color.

## Copywriting Contract

Clean operator/service/aggregator framing (D-09, HOST-03 direction). NO "booth" / "attendee" / "demo" wording in any new Phase 2 string. Sentence case for buttons and copy; the participant is addressed directly ("you"/"your keys").

| Element | Copy |
|---------|------|
| Primary CTA | `Join` (per-row) ‚Üí confirm `Join cohort` (inline identity step) |
| Empty state heading | `No open cohorts right now` |
| Empty state body | `This service isn't advertising any cohorts right now. Check back soon.` |
| Error state | `Can't reach this service. We couldn't load its cohort directory ‚Äî retrying automatically‚Ä¶` |
| Destructive confirmation | none this phase ‚Äî no destructive participant action. Leaving: `Leave cohort` returns to the directory with no confirmation dialog (a not-yet-full seat is reclaimed by the service's existing TTL). |

Full string set for this phase:

**Service-identity header (D-02, anonymous)**
- Service origin line (Display/Heading): the service origin (`window.location.host`), rendered as the page's focal heading.
- Service-online indicator (good `StatusDot`, from `PublicStatus.tsx`): `Service online`.
- Active-network chip: `{activeNetwork}` (reuse the existing network chip, incl. the mainnet `{label} ¬∑ REAL FUNDS` bad-tone variant).
- Open-cohort count: `{n} open cohorts` ‚Äî when `n === 0`: `No open cohorts right now`.

**Directory (browse) list (PART-01, D-08/D-09/D-10)**
- Section title (`SectionTitle`): `Open cohorts`
- Per-row fields: beacon-type chip with gloss, active-network chip, seats, co-sign threshold, status badge, cohort id (`CopyField`), `Join` button. Newest-advertised first; no sort/filter controls (D-10).
- Beacon-type chip + gloss (D-08): `CAS ¬∑ content-addressed` / `SMT ¬∑ sparse Merkle tree`.
- Seats (D-08): `{joined}/{capacity} seats` with caption `{capacity - joined} open` (when full: `Full`).
- Co-sign threshold (D-08): `Co-sign: {threshold}-of-{threshold}` with an optional caption `all signers required` (n-of-n).
- Status labels (D-09): `Open` / `Filling` / `Collecting updates` / `Full`. The raw protocol phase may appear on hover (`title=`) or as a small `Mono` if useful, never as the primary label.
- Cohort id (`CopyField` label): `Cohort ID`.
- Row join action: `Join`. Disabled on rows shown `Full` or past the open phases (belt-and-suspenders; the server is authoritative, D-06).

**Inline identity step at Join (PART-02, D-03/D-04)**
- Panel heading (`SectionTitle`/Heading): `Choose an identity to join`
- Context line (Body): `Joining cohort {shortCohortId} ‚Äî {joined}/{capacity} seats, {statusLabel}.`
- Option A (default, generate new KEY identity): label `Generate a new identity` / help `We'll generate a new did:btcr2 key identity in your browser.`
- Option B (import existing / EXTERNAL x1): label `Import an existing identity` / help `Bring a did:btcr2 identity you already control (including an x1/EXTERNAL identity).`
- Key-custody note (Body, always visible, D-04): `Your keys stay in this browser. This service never sees your private key.`
- Confirm button (primary): `Join cohort`
- Confirm in-progress: `Joining‚Ä¶`
- Cancel button (ghost): `Cancel` (returns to the directory without minting anything if generation hasn't run ‚Äî no key material is created behind the participant's back, D-04)

**Join outcomes (D-06/D-12, deterministic ‚Äî no dead spinners)**
- Seated success (transient, good tone): `Seated ‚Äî you're in.`
- Seated resting confirmation (D-11): heading `You're seated in cohort {shortCohortId}` / body `{joined}/{capacity} seats ¬∑ {statusLabel}. When this cohort fills, co-signing begins below.`
- Failed join ‚Äî full/closed during the poll window (D-06, return to browse, bad tone): `That cohort just filled or closed. Pick another from the directory.`
- Failed join ‚Äî generic error (bad tone): `Couldn't join that cohort. It may have closed ‚Äî pick another from the directory.`

**Seated + leave (D-11/D-15)**
- Leave button (ghost): `Leave cohort`
- After leave: return to the directory landing (no confirmation dialog).
- The seated card links into the reused tail: the existing `RegisterPanel`/`PublishPanel`/`ResolvePanel` render below the seated confirmation unchanged (D-11); no new copy for the tail this phase (Phase 3 owns it).

**Non-happy directory states (D-12)**
- Directory empty (benign, keep polling): heading `No open cohorts right now` / body `This service isn't advertising any cohorts right now. Check back soon.` (reuses Phase 1's public-directory copy verbatim).
- Service unreachable / directory fetch error (distinct, auto-retry, bad tone): heading `Can't reach this service` / body `We couldn't load this service's cohort directory. Retrying automatically‚Ä¶`
