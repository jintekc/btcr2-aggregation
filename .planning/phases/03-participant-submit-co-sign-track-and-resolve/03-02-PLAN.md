<objective>
Add the one small server piece Phase 3 needs: a minimal public anchor-status read backed by a bounded, per-service retained map fed by the existing `BeaconBroadcaster`. This is the PART-04 tracking source the browser polls (D-20/D-21/D-22) - anchor facts are public chain data, so the read is anonymous and must not weaken the gated telemetry posture (ADR 0015).

Purpose: Surface the transient broadcast lifecycle as a pollable last-known fact without a new SSE, without chain I/O on a public route, and without exposing anything non-public.
Output: `anchor-state.ts` (bounded map + DTO + factory), a public `GET /v1/anchor/:cohortId` route, and createService wiring.
</objective>

## Phase Goal (user story, derived from the ROADMAP Phase 3 goal)

**As a** participant who joined a cohort by choice, **I want to** submit my DID update, take part in the cohort's n-of-n MuSig2 co-signing round, track the anchor, and resolve my updated DID, **so that** I complete the real aggregation lifecycle from the cohort I discovered, not a standalone demo stepper.

> This plan is the thin public read the Wave 3 tracking UI (03-06) polls; it adds no user-visible surface on its own (brownfield-rewire reality).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| anonymous client -> GET /v1/anchor/:cohortId | Untrusted public read of a per-cohort anchor fact |
| BeaconBroadcaster -> retained map | Server-internal transient frames folded into last-known state |
| public read block vs operatorAuth block | The mount site decides whether the route inherits the session guard |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-02-01 | Information disclosure | anchor DTO | medium | mitigate | DTO carries only public chain facts (state/txid/explorerUrl) + enabled; broadcast-failure reason kept generic; no member DIDs/keys. |
| T-03-02-02 | Information disclosure | unknown-cohort answer | low | mitigate | Unknown cohortId returns `{state:'none'}`, identical for never-existed and evicted (no existence oracle). |
| T-03-02-03 | Denial of Service | retained map growth | medium | mitigate | Bounded ~24 with oldest-first eviction; reads O(1); no chain I/O on the read path. |
| T-03-02-04 | Elevation of privilege / Access control | route mount site | high | mitigate | Route mounted OUTSIDE the operatorAuth block; /dashboard/* and /v1/operator/* gating byte-untouched; read-only, no body. |
| T-03-02-05 | Denial of Service | esplora-on-public-read | high | mitigate | The read never touches esplora; it returns retained broadcaster state, keeping the hermetic default chain-free. |

No new packages are installed in this plan (zero-install phase); supply-chain checkpoint not applicable.
