<objective>
Close the one remaining Phase 3 gap (03-VERIFICATION.md Truth 8, status gaps_found 7/8): a broadcast-but-unconfirmed anchor (`enabled: true, state: 'broadcast'`) is still narrated as "Anchored"/"anchored" on the participant's cohort page while AnchorSubSteps, in the very same view, correctly renders "Confirmed: pending" beneath it. This is the SAME mode-honesty contradiction class that plans 03-07 and 03-08 closed for other sub-cases (`state: 'none'`, `state: 'failed'`, and the pre-first-read null window), persisting in the one sub-case those rounds did not scope, and it is currently pinned as intended behavior by participant.spec.ts.

The defect lives in three pure/derived surfaces, all fed by the same `state === 'broadcast' -> anchored` conflation:

- `anchorSummaryState` (participant.ts:701-703) maps `state === 'confirmed' || state === 'broadcast'` to 'anchored'. This drives the StageTimeline final-row label (via the `summary === 'anchored'` gate added by 03-08) and the CompletionSummary narration paragraph. A broadcast tx therefore shows a "Signed and anchored on {netLabel}." paragraph and an "Anchored" timeline header.
- `deriveStage` (participant.ts:628) maps `state === 'confirmed' || state === 'broadcast'` to the 'anchored' Stage. This drives the persistent "Your cohort · Anchored" header chip (App.tsx:96, BrowseView.tsx:132) and marks the timeline journey done while the tx is still unconfirmed.
- CompletionSummary's `anchored` boolean (CompletionSummary.tsx:72) mirrors the same `confirmed || broadcast` condition, so the completion-card heading reads "Anchored" for a broadcast tx.

At the same moment, AnchorSubSteps (StageTimeline.tsx:73-79) computes `confirmed = anchor.state === 'confirmed'` independently, so it (correctly) says "Confirmed: pending" beneath all of the above. The fix is the one sketched in 03-REVIEW.md WR-02: reserve the 'anchored' value for `state === 'confirmed'` in every selector/boolean that produces it, and route `state === 'broadcast'` into the existing honest 'broadcasting' narration (and the 'signed' Stage), whose copy is already correct.

Purpose: make the whole completion view, including the persistent chip, narrate one honest anchor state for the broadcast-but-unconfirmed window, closing the last reachable internal contradiction so PART-04's "track...anchor status" clause and the phase's D-07 mode-honesty through-line hold for EVERY reachable anchor state.
Output: a corrected `anchorSummaryState` and `deriveStage` (both reserve 'anchored' for `state === 'confirmed'` only), an aligned CompletionSummary `anchored` boolean, and updated spec pins that assert the honest values ('broadcasting' / 'signed') for the broadcast state. StageTimeline.tsx needs no source edit: it already reads the corrected selector, so its label auto-corrects.
</objective>
