# Phase 3 - UI Design Contract

> Visual and interaction contract for the participant submit / co-sign / track / resolve flow. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase REWIRES the existing signing/resolve machinery into the browse->join path (Phase 2) and RETIRES the standalone linear stepper. It introduces NO new design language: it extends the shipped Tailwind v4 `@theme` tokens and the `packages/web/src/ui/primitives.tsx` component set already used by the directory, join, and operator surfaces. Every token, color, and type role below is DETECTED from the shipped system, not invented.

## Spacing Scale

The shipped system uses Tailwind's default 4px base scale. Declared values in active use:

| Token | Value | Usage |
|-------|-------|-------|
| 3xs | 2px | Pill vertical padding only (`Badge` `py-0.5`) - legacy primitive exception |
| 2xs | 4px | Tight stacks (`space-y-1`, dot-to-label gaps) |
| xs | 6px | Field label-to-control gap (`space-y-1.5`) |
| sm | 8px | Compact element spacing, flex gaps (`gap-2`) |
| md | 12px | Row/inset padding, chip gaps (`px-3`, `gap-3`) |
| lg | 16px | Default card padding, panel gaps (`p-4`, `gap-4`) |
| xl | 24px | Metric-column separation, section gaps (`gap-6`) |
| 2xl | 32px | Major stage/section breaks |
| 3xl | 48px | Page-level vertical rhythm |

Exceptions: `Badge` pills use `px-2.5 py-0.5` (10px / 2px) and `CopyField`/`Mono` use half-step paddings from the shipped primitives. Reuse the primitives as-is - do NOT re-pad them. New stage-timeline containers use `p-4` cards with `gap-4`/`space-y-3` internal rhythm to match `CohortRow`.

## Typography

Constrained scale for all Phase 3 surfaces: exactly 4 sizes, exactly 2 weights. Detected from shipped usage, then collapsed to the contract limits.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 | 1.45 (Tailwind `leading-normal`) |
| Label | 12px (`text-xs`, uppercase, `tracking-[0.14em]`) | 600 | 1.2 |
| Heading | 20px (`text-xl`) - stage/panel/section title inside the cohort page | 600 | 1.25 |
| Display | 30px (`text-3xl`) - service-identity / page title (reused from Phase 2 header, not re-created) | 600 | 1.2 |

Weights: 400 (body, mono values) and 600 (labels, buttons, headings). No other weight is permitted on Phase 3 surfaces.

Normalizations that make the scale hold (small, accepted visual deltas applied in this phase):

- The 10px micro-caption variants (`text-[0.65rem]`, `text-[0.6rem]`) are dropped for Phase 3 surfaces: every micro-caption uses the 12px Label role. New components never introduce sub-12px sizes.
- Mono values (DID, cohort id, beacon address, txid, raw JSON) render at the Label size, 12px (`text-xs`), weight 400, with `truncate`; the `Mono` primitive's `text-[0.8rem]` is normalized to `text-xs` (a sub-1px delta). Full values stay reachable through `CopyField` or the raw-detail expander.
- `Badge`'s internal `font-medium` (500) is folded into `font-semibold` (600) so the app carries exactly two weights; the visual delta on pills is minor and accepted.

Use `tabular-nums` for seat counts / co-sign figures / elapsed indicators (matches `CohortRow`).

## Color

Palette is a dark slate canvas with bitcoin-orange as the single accent (oklch tokens from `@theme`). The 60/30/10 split maps directly onto the shipped roles.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-canvas` `oklch(0.17 0.015 260)` (+ `--color-ink` `oklch(0.95 0.01 260)` text, `--color-muted`, `--color-faint`) | Page background, inset input/copy surfaces |
| Secondary (30%) | `--color-surface` `oklch(0.21 0.018 260)`, `--color-surface-2` `oklch(0.26 0.02 260)`, borders `--color-edge` / `--color-edge-strong` | Cards, stage panels, expander, ghost buttons |
| Accent (10%) | `--color-accent` `oklch(0.74 0.16 55)` on `--color-accent-ink` | See reserved list below |
| Destructive | `--color-bad` `oklch(0.68 0.2 25)` | Destructive actions + honest-error/mainnet REAL-FUNDS marks only |

Semantic status tokens (NOT part of the 10% accent budget; used as low-saturation `/15` fills + `/40` borders on `Badge`/`StatusDot`): `--color-good` `oklch(0.78 0.16 155)` (seated / signed-complete / reflected), `--color-warn` `oklch(0.82 0.15 90)` (waiting / in-progress / submit-window urgency / degraded), `--color-bad` (terminal failure / mismatch / mainnet).

**Accent reserved for:** the single primary action button at the active stage (`Submit my DID update`, `Resolve again`, `Browse cohorts`), the active/live stage's `StatusDot` (accent tone, `pulse` while live), and the "You're in this cohort" row's `View cohort` primary button. Accent is NEVER used for: pending/complete stage dots, semantic status badges, expander chrome, secondary/ghost actions (`Leave`, `Start over`, copy buttons), or body text. One accent focus per view, matching Phase 2's "accent stays scarce" rule (`CohortRow` doc comment).

**Primary visual anchor on cohort-page load:** the stage timeline, with the active stage highlighted (good-tone completed dots, pulsing accent dot on the active stage, dimmed pending stages). The single accent CTA renders inline at the active stage and inherits focus from the timeline; it never competes with the timeline as a separate page-level anchor. The "Your cohort" link and the technical-detail expander are secondary chrome.
