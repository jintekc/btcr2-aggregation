# Phase 3 - UI Design Contract

> Visual and interaction contract for the participant submit / co-sign / track / resolve flow. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase REWIRES the existing signing/resolve machinery into the browse->join path (Phase 2) and RETIRES the standalone linear stepper. It introduces NO new design language: it extends the shipped Tailwind v4 `@theme` tokens and the `packages/web/src/ui/primitives.tsx` component set already used by the directory, join, and operator surfaces. Every token, color, and type role below is DETECTED from the shipped system, not invented.

## Spacing Scale

The shipped system uses Tailwind's default 4px base scale. Declared values in active use:

| Token | Value | Usage |
|-------|-------|-------|
| 3xs | 2px | Pill vertical padding only (`Badge` `py-0.5`) - legacy primitive exception |
| 2xs | 4px | Tight stacks (`space-y-1`, dot-to-label gaps) |
| xs | 6px | Field label-to-control gap (`space-y-1.5`) |
| sm | 8px | Compact element spacing, flex gaps (`gap-2`) |
| md | 12px | Row/inset padding, chip gaps (`px-3`, `gap-3`) |
| lg | 16px | Default card padding, panel gaps (`p-4`, `gap-4`) |
| xl | 24px | Metric-column separation, section gaps (`gap-6`) |
| 2xl | 32px | Major stage/section breaks |
| 3xl | 48px | Page-level vertical rhythm |

Exceptions: `Badge` pills use `px-2.5 py-0.5` (10px / 2px) and `CopyField`/`Mono` use half-step paddings from the shipped primitives. Reuse the primitives as-is - do NOT re-pad them. New stage-timeline containers use `p-4` cards with `gap-4`/`space-y-3` internal rhythm to match `CohortRow`.

## Typography

Constrained scale for all Phase 3 surfaces: exactly 4 sizes, exactly 2 weights. Detected from shipped usage, then collapsed to the contract limits.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 | 1.45 (Tailwind `leading-normal`) |
| Label | 12px (`text-xs`, uppercase, `tracking-[0.14em]`) | 600 | 1.2 |
| Heading | 20px (`text-xl`) - stage/panel/section title inside the cohort page | 600 | 1.25 |
| Display | 30px (`text-3xl`) - service-identity / page title (reused from Phase 2 header, not re-created) | 600 | 1.2 |

Weights: 400 (body, mono values) and 600 (labels, buttons, headings). No other weight is permitted on Phase 3 surfaces.

Normalizations that make the scale hold (small, accepted visual deltas applied in this phase):

- The 10px micro-caption variants (`text-[0.65rem]`, `text-[0.6rem]`) are dropped for Phase 3 surfaces: every micro-caption uses the 12px Label role. New components never introduce sub-12px sizes.
- Mono values (DID, cohort id, beacon address, txid, raw JSON) render at the Label size, 12px (`text-xs`), weight 400, with `truncate`; the `Mono` primitive's `text-[0.8rem]` is normalized to `text-xs` (a sub-1px delta). Full values stay reachable through `CopyField` or the raw-detail expander.
- `Badge`'s internal `font-medium` (500) is folded into `font-semibold` (600) so the app carries exactly two weights; the visual delta on pills is minor and accepted.

Use `tabular-nums` for seat counts / co-sign figures / elapsed indicators (matches `CohortRow`).

## Color

Palette is a dark slate canvas with bitcoin-orange as the single accent (oklch tokens from `@theme`). The 60/30/10 split maps directly onto the shipped roles.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-canvas` `oklch(0.17 0.015 260)` (+ `--color-ink` `oklch(0.95 0.01 260)` text, `--color-muted`, `--color-faint`) | Page background, inset input/copy surfaces |
| Secondary (30%) | `--color-surface` `oklch(0.21 0.018 260)`, `--color-surface-2` `oklch(0.26 0.02 260)`, borders `--color-edge` / `--color-edge-strong` | Cards, stage panels, expander, ghost buttons |
| Accent (10%) | `--color-accent` `oklch(0.74 0.16 55)` on `--color-accent-ink` | See reserved list below |
| Destructive | `--color-bad` `oklch(0.68 0.2 25)` | Destructive actions + honest-error/mainnet REAL-FUNDS marks only |

Semantic status tokens (NOT part of the 10% accent budget; used as low-saturation `/15` fills + `/40` borders on `Badge`/`StatusDot`): `--color-good` `oklch(0.78 0.16 155)` (seated / signed-complete / reflected), `--color-warn` `oklch(0.82 0.15 90)` (waiting / in-progress / submit-window urgency / degraded), `--color-bad` (terminal failure / mismatch / mainnet).

**Accent reserved for:** the single primary action button at the active stage (`Submit my DID update`, `Resolve again`, `Browse cohorts`), the active/live stage's `StatusDot` (accent tone, `pulse` while live), and the "You're in this cohort" row's `View cohort` primary button. Accent is NEVER used for: pending/complete stage dots, semantic status badges, expander chrome, secondary/ghost actions (`Leave`, `Start over`, copy buttons), or body text. One accent focus per view, matching Phase 2's "accent stays scarce" rule (`CohortRow` doc comment).

**Primary visual anchor on cohort-page load:** the stage timeline, with the active stage highlighted (good-tone completed dots, pulsing accent dot on the active stage, dimmed pending stages). The single accent CTA renders inline at the active stage and inherits focus from the timeline; it never competes with the timeline as a separate page-level anchor. The "Your cohort" link and the technical-detail expander are secondary chrome.

## Copywriting Contract

All copy is mode-honest per D-07/D-23/D-25/D-29 (truthfulness over polish is the phase through-line). No "booth"/"attendee" wording. No em-dash character anywhere. Copy branches on the anchor read's `enabled` flag (live vs hermetic).

| Element | Copy |
|---------|------|
| Primary CTA (submit) | `Submit my DID update` |
| Submit preview lead | `This appends this cohort's beacon service to your DID document. Review the update below, then submit it to the round.` |
| Beacon-commitment consent line (D-14, hermetic) | `Submitting also co-signs this cohort's aggregated update commitment. This service runs without on-chain broadcast, so nothing is published to Bitcoin.` |
| Beacon-commitment consent line (D-14, live) | `Submitting also co-signs the beacon transaction that anchors this cohort's aggregated update commitment at beacon address {addr}. This is a real broadcast on {network}.` |
| Submit-window urgency (D-13) | Stage heading `Your update is needed` + tab title `(!) Submit your update`. Body: `The cohort is waiting on your submission before it can co-sign.` |
| Stage labels (D-01) | `Waiting for seats` / `Seated` / `Submit update` / `Co-signing` / `Signed` (hermetic) or `Anchored` (live, with txid) / `Resolved` |
| Elapsed indicator (D-05) | `Active for {mm:ss}` (quiet, no countdown, no duration promise) |
| Signed-complete, hermetic (D-07) | `Signed. This no-broadcast service does not publish to Bitcoin, so there is no on-chain anchor to show.` |
| Anchored, live (D-22) | `Anchored` -> stage sub-steps `Signed` / `Broadcast` (txid + `View on explorer`) / `Confirmed` |
| k-of-n fallback outcome (D-23) | `The cohort anchored via the k-of-n fallback path with {k} of {n} signatures. Your update was {included / not included}.` |
| Round-trip: reflected (live) | `Your update is reflected. The resolved DID document now lists this cohort's beacon service (version {N}).` |
| Round-trip: hermetic-genesis (expected, NOT an error) | `Resolved to the genesis document. This no-broadcast service has no on-chain signal to discover, so your co-signed update lives in the downloadable sidecar/artifacts below.` |
| Round-trip: mismatch (live) | `Your update was not found in the resolved document yet. If the anchor just confirmed, the resolver may still be indexing. Try Resolve again.` |
| Empty state heading | `No cohort joined yet` |
| Empty state body | `Browse this service's open cohorts and join one to submit a DID update.` + `Browse cohorts` CTA (returns to the directory landing) |
| Error: service unreachable (D-24) | Heading `Can't reach this service` / body `We'll keep trying to reconnect. Your place in the cohort is unaffected as long as this tab stays open.` (quiet auto-retry, stages freeze) |
| Error: cohort stalled (D-13 / Finding 2) | `The cohort ended. It stalled waiting for all members to submit their updates.` + `Back to cohorts` |
| Error: terminal, specific (D-25) | best-effort: `The cohort ended: phase timed out.` / `... : the cohort is no longer available.` / `... : the signing round errored.` / `... : your seat was lost.` |
| Error: terminal, unknown (D-25 fallback) | `The cohort ended and this service didn't say why.` + `Back to cohorts` |
| Cooperative non-inclusion (D-10, non-error) | `The cohort proceeded without your update. {reason}. Here is how the cohort finished:` (keeps reporting the cohort's anchor result) |
| Keep-tab-open note (D-08) | `Keep this tab open. Refreshing loses your seat, and this service does not save your session yet.` |
| Identity / key-custody note (D-06) | `You hold this key in your browser only. It is never sent to the service.` |
| Directory row: joined (D-04) | Row state `You're in this cohort` + primary `View cohort` action (replaces Join; Join disabled on all other rows while seated) |
| "Your cohort" persistent link (D-03) | `Your cohort · {current stage label}` (label + `StatusDot` update live) |
| In-flight directory rows (D-26) | Non-joinable status `In progress` |
| Completion export offer (D-30) | `Download sidecar (resolver artifacts)` + body `Anyone can use this file to resolve your updated DID without contacting this service.` |
| Primary CTA (resolve rerun) | `Resolve again` |
| Terminal / summary CTA | `Browse cohorts` |

### Destructive Confirmations

| Element | Copy |
|---------|------|
| Leave (D-09, only while waiting for seats) | `Leave`: `Leave this cohort? You can browse and join another. Once co-signing starts you are committed through anchoring.` |

## UI Considerations

> Shape-rooted UI state coverage. Empty/error COPY lives in the Copywriting Contract above; this section covers state coverage and references those rows.

Applicable state considerations resolved: 8 covered, 2 backstop, 0 unresolved.

| Category | Element(s) | Status | Resolution / Reason |
|----------|------------|--------|---------------------|
| empty | cohort page with no join / list-collection | ✅ covered | Renders the documented `No cohort joined yet` empty state with a `Browse cohorts` CTA back to the directory landing |
| loading | active stage (waiting-for-seats, co-signing, resolving) | ✅ covered | Active stage shows the quiet `Active for {mm:ss}` indicator + `StatusDot` `pulse`; future stages dimmed as pending (D-05); resolve shows brief `resolving...` retries gated on `enabled` (D-28) |
| error | service unreachable (transient) | ✅ covered | Distinct `Can't reach this service` banner with quiet auto-retry; stages freeze honestly, never a terminal by itself (D-24) |
| error | terminal failure (stall / timeout / vanished / seat lost / unknown) | ✅ covered | Best-effort specific reason with the honest `didn't say why` fallback; lands on the page with `Back to cohorts` (D-25, Finding 2) |
| populated | full stage timeline | ✅ covered | Full timeline upfront, active stage highlighted, completed stages good-tone, pending dimmed (D-01/D-05) |
| partial | k-of-n fallback outcome + cooperative non-inclusion | ✅ covered | Explicit fallback-path copy stating k of n and inclusion (D-23); non-inclusion rendered as a distinct non-error outcome that still reports the cohort's anchor result (D-10) |
| overflow | long DID / txid / beacon address / cohort id | ✅ covered | `Mono` `truncate` + `CopyField` guarantee the full value is copyable without breaking layout (shipped primitives) |
| zero-one-many | activity log entries + seat counts | ✅ covered | Timestamped activity log accumulates inside the technical-detail expander (D-27); seat/co-sign figures use `tabular-nums`; aggregate counts about others only, never member identities (D-08) |
| long-text | raw signed-update JSON + full resolved DID document | 🧪 backstop | Held behind the raw-detail expander (collapsed by default); UI-state test must confirm large JSON does not overflow the card and the expander scrolls |
| loading | anchor sub-step transitions (Signed -> Broadcast -> Confirmed, live) | 🧪 backstop | Anchor poll on ~5s cadence, freeze at first confirmation (D-22); backstop test confirms the sub-step chevron renders each state and stops polling at `confirmed`/`failed` |
