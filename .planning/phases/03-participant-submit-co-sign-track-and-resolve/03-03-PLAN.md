<objective>
Widen the public directory to list in-flight cohorts as honest non-joinable "In progress" rows (D-26) so a service with a cohort mid-signing still looks alive to a stranger, WITHOUT changing the joinable count or the Phase 2 join gate. This is the directory half of the PART-04 tracking surface; the client renders the label (03-06).

Purpose: Make the directory reflect the whole live set (pre-signing + signing) while keeping "joinable" and "open count" exactly Advertised-only.
Output: A display-only in-flight phase tier in `directory()`, a joinable-only `status()` count, and specs pinning both.
</objective>

## Phase Goal (user story, derived from the ROADMAP Phase 3 goal)

**As a** participant who joined a cohort by choice, **I want to** submit my DID update, take part in the cohort's n-of-n MuSig2 co-signing round, track the anchor, and resolve my updated DID, **so that** I complete the real aggregation lifecycle from the cohort I discovered, not a standalone demo stepper.

> This plan widens the public directory read the tracking UI (03-06) and the store's post-seat cohort-gone detection (03-04) rely on; it adds no user-visible surface on its own.

## Artifacts this phase produces (this plan)

- A display-only in-flight phase set (e.g. `IN_FLIGHT_PHASES`) in `operator-cohorts.ts`, kept distinct from `OPEN_PHASES`.
- A joinable-only open-count path inside `status()` (no longer `directory().length` if `directory()` widens).
- New spec cases in `operator-cohorts.spec.ts`.
